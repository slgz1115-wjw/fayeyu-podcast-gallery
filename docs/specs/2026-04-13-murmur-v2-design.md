# Murmur v2 — 设计文档

**日期**：2026-04-13
**状态**：已确认，待实现

## 概述

将当前的 Murmur v1（承袭自 Podcast Gallery 的 SaaS 式后台界面）重构为一个**个人知识工作台**风格的 Web 应用：无侧边栏、大封面、深色沉浸氛围、以「来源流」为核心的交互模型。

## 核心设计决策

### 1. 视觉氛围：深夜书房

- 主色：深黑 `#0d0d0d`（比 v1 的 `#1A1A1A` 更暗）
- 文本：暖米白 `#f5f0eb`（不是纯白，避免刺眼）
- 强调色：橙色 `#FF6A00`（点缀暖光，不是全局主色）
- 紫色 `#A78BFA` 用于飞书/对谈类笔记的标记
- Logo 字体：Georgia/衬线 + 斜体 + 超细字重 + 宽字距
- 背景装饰：顶部 radial gradient 暖橙光晕，模拟台灯

### 2. 页面结构：无侧栏全纵向

```
┌─────────────────────────────────────┐
│          HERO                        │
│   ╭────────╮                         │
│   │ murmur │  ← 大号艺术字 Logo      │
│   ╰────────╯                         │
│   低语 · knowledge softly gathered    │
│                                     │
│   [🎙 播客 3]  [📄 飞书]  [✍️ 笔记] │  ← 来源 pills
├─────────────────────────────────────┤
│       Library | Graph | Skills      │  ← 居中 tab
├─────────────────────────────────────┤
│   [全部] [🎙] [📄] [✍️] [⭐]         │  ← 筛选
│                                     │
│   ▌ 笔记卡片                         │
│   ▌ 笔记卡片                         │
│   ▌ 笔记卡片                         │
│                                     │
└─────────────────────────────────────┘
```

**移除的元素**：
- 侧边栏完全去掉
- 独立的 Podcasts tab（合并到 🎙 pill 的抽屉）
- 独立的 Starred tab（改为 Library 的筛选器）
- Inbox tab（并入 Library 主视图 + 来源 pills）

**保留的元素**：
- Library / Graph / Skills 三个核心 tab
- 抽屉式笔记详情（从右侧滑出）
- 术语高亮 + 弹出卡片
- 知识图谱分类树

### 3. 来源 Pills 交互

点击 Hero 里的 pill → 从右侧滑出专属抽屉面板：

- **🎙 播客 pill**（显示新剧集数量）
  - 抽屉内容：订阅管理 + 待处理队列 + 添加新订阅 + 喂入单集
  - 相当于原来的 Podcasts tab 内容
- **📄 飞书 pill**
  - 抽屉内容：搜索输入框 + 实时搜索结果（调 lark-cli）
  - 选择文档 + 选择 Skill + 确认添加
- **✍️ 笔记 pill**
  - 抽屉内容：大号 textarea + 标题输入 + Skill 选择
  - 用于粘贴已整理好的 markdown 内容

### 4. Library 视图

- **统一显示**所有来源的完成笔记
- 卡片设计：左侧 3px 色条 + 来源 tag + 标题 + 摘要
  - 橙色色条 = 播客
  - 紫色色条 = 飞书/对谈
  - 灰色色条 = 粘贴笔记
- 筛选栏：`全部 | 🎙 | 📄 | ✍️ | ⭐ 收藏`
- 点击卡片 → 右侧抽屉展开 markdown 全文

### 5. Skills 系统

三个核心 Skill（已入库，完整长度版本）：

| Skill | Category | 长度 | 用途 |
|------|---------|------|------|
| 播客三模块提炼 | podcast | 1026 字符 | 播客专用，不变 |
| 高密度对谈逻辑梳理 | transcript | 2949 字符 | 会议/访谈/讨论，产出四层内容 |
| 结构化笔记·概念提取 | article | 905 字符 | 已整理好的 markdown，仅提取概念 |

**关键**：这次 `高密度对谈逻辑梳理` 和 `结构化笔记·概念提取` 都会输出 `## 专业名词解释词典` 段落，前端会解析这些术语进入知识图谱（与播客的 Module C 一致）。

Skill prompt 源文件保存在 `podcast-hub/skills/*.md`，支持 git 版本管理。

### 6. 数据库

**不需要 schema 变更**。现有的 `skills` / `notes` / `episodes` 表结构足够支撑新设计。

## 实现范围

**本次要改的文件**：
- `public/index.html` — 完整重写前端（HTML/CSS/JS）
- `public/logo.svg` — 重新设计 logo（或直接用艺术字文本，不用 svg）
- `server.js` — 小改：可能需要补 `/api/notes/:id/process` 端点用于触发笔记的 Skill 提炼

**不动的文件**：
- `pipeline.js` — 播客 pipeline 不变
- `transcribe.py` — 不变
- 数据库 schema — 不变

**数据迁移**：零迁移，现有 245 剧集数据完整保留。

## 不在本次范围内

- 知识图谱的 Obsidian 化（双向链接图可视化）
- URL 的自动抓取管道
- 本地文件上传管道（docx/pdf 解析）
- 飞书文档内容的实际拉取（本次只做搜索+存入，拉取内容的 pipeline 留到下一步）

这些都是独立功能，后续迭代。

## 验收标准

1. 打开 http://localhost:3456 看到深色 Hero + 艺术字 Logo
2. Hero 里有三个来源 pills，点击各自展开抽屉面板
3. 🎙 pill 里能看到所有订阅和待处理剧集，跟原 Podcasts tab 功能一致
4. 📄 pill 里能搜索飞书文档
5. ✍️ pill 里能粘贴 markdown 并选 Skill 存入
6. Library 展示所有笔记，筛选器能用
7. Graph 和 Skills 两个 tab 保留现有功能
8. 已有的 4 条播客笔记在新界面里正常显示，点击能看到全文
