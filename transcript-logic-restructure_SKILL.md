# 底层逻辑学习：文字记录逻辑梳理

## 这个 skill 解决什么问题

口语化的讨论记录（会议录音转写、访谈逐字稿、飞书妙记等）往往信息密度很高，但逻辑线索被口语表达、重复、插科打诨、跑题等淹没。直接读原文很难提取其中的推理链条、经验教训和逻辑闭环。

简单的"总结"会丢失大量细节和推理过程。这个 skill 的核心理念是：**先抽离底层知识，再梳理讨论脉络，最后总结**——产出三层内容，从最抽象的可迁移知识到最具体的讨论细节，服务于深度学习。

## 产出物

一份文档，包含三层内容（按顺序排列）：

1. **底层知识体系**（第〇章）：从讨论中抽离的、不依赖具体语境的可穿越周期的知识框架
2. **逻辑梳理版**（正文章节）：按讨论顺序，篇幅与原文基本一致的逻辑重组版本
3. **总结**（可选，或单独文件）：基于梳理版的精华提炼

## 第一步：理解原文

### 获取内容

根据用户提供的来源获取原始文字记录：
- 如果是飞书文档/妙记，使用 lark-doc 或 lark-minutes skill 获取
- 如果用户直接粘贴了文本，直接使用
- 如果是本地文件，读取文件内容

### 通读标注

通读全文时，在脑中建立以下索引：

1. **说话人识别**：辨认每个说话人的角色和立场。飞书妙记中的 `<mention-user>` 和 `<text color="gray">说话人 N</text>` 标签是识别依据。
2. **话题边界**：标记话题的起止位置。
3. **逻辑链条**：识别每个话题内的推理链——前提→推理→结论，以及引用的经验、案例和类比。
4. **可穿越周期的知识点**：标记那些不依赖于具体人、组织、时间的底层逻辑——这些将在第〇章中提炼。
5. **高价值内容 vs 噪音**：区分有实质信息密度的内容和纯噪音。

## 第二步：产出底层知识体系（第〇章）

这是本 skill 的核心差异化步骤。目标是从讨论中**抽离出可穿越周期的知识框架**——完全不以这次具体谈话的内容为转移，可以独立存在、被迁移到其他场景的知识体系。

### 什么是"可穿越周期的知识"

判断标准：如果把这段知识从这篇文档中单独拿出来，放到完全不同的行业/组织/时间背景下，它仍然成立吗？如果是，就是可穿越周期的知识。

典型类型：
- **模型/框架**：一个可以用公式或结构图表达的思考模型（如"社会债务 = 价值 - 价格"）
- **机制**：一个可复用的因果机制（如"初始优势→信心→更多投入→更强优势"的正向循环）
- **悖论/张力**：一个在多种场景中反复出现的矛盾（如"组织标准化 vs 野蛮生长能力"的张力）
- **原则**：一个经过论证的行动原则（如"真实场域培养优于模拟环境培养"）

### 写作要求

每个知识框架应包含：

1. **核心命题**：用一句话概括这个框架的核心洞察
2. **模型结构**：用图表/公式/分层结构清晰呈现框架的逻辑骨架
3. **关键要素**：展开解释框架中的每个关键节点，说明为什么它重要
4. **适用场景**（可选）：这个框架在什么场景下可以被复用

### 关键原则

- **去人名化**：不出现讨论中的具体人名（名人/公众人物的案例除外），用角色描述替代（如"讨论者"、"组织创始人"、"一位成员"）
- **去个人经历**：不写某个人的具体个人经历（名人案例除外），而是从个人经历中抽象出普遍规律
- **完全独立**：这一章的内容应该可以独立阅读，不需要读后面的讨论脉络也能理解
- **可迁移**：读者可以直接把这些框架应用到自己的场景中

### 结构模板

```markdown
## 〇、底层知识体系（从本次讨论中抽离的可穿越周期的逻辑）

以下知识体系从本次讨论中提炼，但不依赖于任何具体的人、组织或时间背景。它们是可迁移、可复用的底层认知框架。

### 框架一：[框架名称]——[一句话核心命题]

**核心命题**：[展开]

**模型结构**：
[用图表/公式/表格呈现]

**关键要素**：
1. ...
2. ...

**适用场景**：[可选]

---

### 框架二：[框架名称]
...
```

## 第三步：产出逻辑梳理版（正文章节）

### 核心原则

逻辑梳理版**不是总结**，它的目标是：

- **篇幅与原文基本一致**——保留所有有信息价值的内容，不做大幅度压缩
- **按讨论顺序组织**——保持原始讨论的时间线脉络，让读者能感受到讨论的展开过程
- **按话题重组**——将散落在时间线各处的同一话题内容聚合在一起
- **理顺推理链条**——将口语中断断续续、被插话打断的推理过程还原为完整的逻辑链
- **去除噪音**——去掉重复、语气词、无关闲聊、跑题段落，但保留有信息量的"题外话"
- **去人名化**——不出现具体人名（名人除外），用角色/代号替代
- **去个人经历**——不写具体个人经历（名人除外），将经历抽象为规律或案例
- **交叉引用底层框架**——在讨论中涉及到第〇章某个框架的地方，标注"（底层逻辑见框架X）"

### 写作要点

**关于保留深度**：
- 当说话人讲了一个完整的逻辑闭环，必须完整保留，不能只留结论
- 当说话人引用了历史经验或案例来论证观点，保留案例的关键细节
- 当多人在一个话题上形成了递进式讨论，保留这个递进过程

**关于去除噪音**：
- 去掉纯粹的口语填充（"对对对"、"嗯"、"就是"、"然后"等）
- 去掉与讨论无关的闲聊
- 去掉重复——同一个意思被说了多遍的，保留表述最清晰的那一遍
- 但注意：有些看起来像闲聊的内容可能包含有价值的类比或隐喻，需要判断

**关于书面化**：
- 将口语转为书面语，但保持表达风格和用词特点
- 口语中的生动比喻和类比可以保留
- 不要加入原文没有的观点或分析

**关于去人名化**：
- 用角色替代具体人名，如"讨论者"、"组织创始人"、"一位资深成员"、"提问者"
- 如果讨论中有多个人的观点需要区分，可以用"A/B/C"或其他不涉及真名的代号
- 名人/公众人物的案例可以保留姓名（如乔布斯、马斯克、马克思等）

## 第四步：产出总结版（可选）

在逻辑梳理版的基础上，产出精炼总结。总结的重点不是"讨论了什么"，而是"得出了什么"。

### 总结结构

```markdown
# [讨论主题] · 总结

> [时间]

---

## 核心议题与关键结论

### 一、[议题名称]
[2-4段概括]

---

## 值得深入学习的逻辑闭环

1. **[闭环名称]**：[简洁的链条描述]
2. ...
```

**"值得深入学习的逻辑闭环"** 提取的不是结论，而是推理模式：
- 完整的因果链条（A→B→C→反馈回A）
- 框架性的思考模型
- 反直觉的洞察及其论证过程
- 历史经验被抽象为规律的过程

## 输出方式

- 保存为 `[主题]_逻辑梳理版.md`（包含第〇章底层知识体系 + 正文梳理 + 末尾总结）
- 保存到用户指定的位置，默认保存到桌面
- 如果原文很长（超过10万字），分多次写入是正常的

## 处理超长文本

飞书妙记等来源的文字记录可能非常长（几万到几十万字）。处理策略：

1. **分段读取**：每次读取300-500行，逐段通读全文
2. **第一遍：建立全局索引**：快速通读，识别所有话题、逻辑链条和可抽离的知识框架
3. **先写第〇章**：在通读完成后，优先产出底层知识体系——这需要全局视野
4. **第二遍：按话题聚合写作**：针对每个话题，回到相关段落细读，写出梳理版
5. **写完即存**：每完成一个大章节就追加写入文件
