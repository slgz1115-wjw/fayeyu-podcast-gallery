# TL;DR + 概念词典提取（两段输出，不重复原文）

## 任务

对下面这份已经整理好的结构化内容，**输出两个段落**，顺序固定：

1. `## TL;DR` —— 核心观点 + 阐述逻辑 + 叙述脉络（300-500 字）
2. `## 专业名词解释词典` —— 可穿越周期的专业术语

**不要重复原文**、不要做总结、不要添加其他任何章节。系统会自动把原文拼接在这两段中间。

## TL;DR 写作要求

放在整个输出最前面，严格按三段结构：

**核心观点**：用 1-3 句话点破文章最根本的论断、判断或结论。不是"本文讨论了 X"，而是"关于 X，核心论断是 Y"。

**阐述逻辑**：用 4-8 句话梳理文章的论证链条，用"因为 A，所以 B；进一步地因为 B，所以 C"的因果结构。不要罗列章节，而是抽象出推理骨架：前提 → 中间论点 → 结论。

**叙述脉络**：用 1-2 句话描述文章是以什么结构展开的（如：时间纵轴→当下横切；问题→分析→建议；案例铺垫→理论提炼；多角度论证→反常识收尾）。

**硬性要求**：
- 总长度 300-500 字
- 是逻辑的抽象，不是摘要的复述
- 不用"本文讨论了"这种套话，直接给判断和结构

---

## 专业名词提取标准

⚠️ **最重要的规则**：只提取**原文中实际出现或实际讨论**的概念。**不要**从下面的分类举例中照搬术语来凑数。如果文中没提「贝叶斯推理」，就绝对不要因为它是"可穿越周期的概念"就加进来。

### 提取范围（只作为分类参考，不是必选项）

只有当以下类型的概念**在原文中出现**时，才提取：
- **学术/理论概念**（文中被命名或使用的理论框架）
- **数学/物理/CS 术语**（文中用作技术论据的）
- **经济/心理/哲学概念**（文中用来支撑论证的）
- **AI/行业术语**（文中讨论的具体技术）
- **社会/组织理论**（文中实际引用的）
- **作者自创或嘉宾自创的概念**（最重要，标注「原文定义」）——如"信任分层"、"附着式增强"、"可穿越周期的知识"等
- **可迁移的论断/框架**（文中明确提出的公式、命题，如"社会债务 = 价值 − 价格"）

### 验证步骤（每个术语都必须通过）

提取前，先在脑中回答：
1. 这个术语在原文哪一段出现？**能指出上下文**
2. 作者是在用这个概念论证什么？不是简单提一下而是实际使用
3. 如果我删除这个术语的定义，读者理解文章是否有障碍？

三个问题有任何一个答不上来，**就不要提取**。

### 不提取

- 公司名、产品名、人物名（除非是理论命名者如"波普尔"且文中明确讨论该理论）
- 原文只是顺便提了一下，没展开讨论的术语
- 过于笼统的词（学习、成功、思考、创新）
- **从示例类别里照搬的概念**——哪怕看起来很"有理论感"，只要文中没有就不要提

### 数量要求

**宁少勿滥**。8-20 个为宜。如果文章很技术化，可到 25 个；如果文章偏叙事/历史（像个人调研报告），能提 5-10 个就很好了，不要为凑 8 个就强行套模板概念。

## 输出格式（严格遵守）

你的整个回答必须严格为以下格式，**不要有前言、不要有收尾、不要重复原文**：

```
## TL;DR

**核心观点**：...

**阐述逻辑**：...

**叙述脉络**：...

---

## 专业名词解释词典

**[术语1]**（English Name）完整定义，说明是什么、为什么重要。

**[术语2]**（English Name 2）定义...

**[术语3]**（没有英文名则省略括号）定义...
```

## 严格要求

- 必须以 `## TL;DR` 开头，中间 `---` 分隔，然后 `## 专业名词解释词典`
- 两个 `##` 标题必须一字不差，它们是下游系统解析锚点
- 每个术语必须用 `**[术语名]**` 格式包围（中括号+双星号）
- 有英文的用中文括号 `（English）`
- 每个术语必须有完整的一句话以上的定义，**不能只列名字**
- 术语之间用空行分隔
- **整个输出只包含这两部分，不允许其他章节**
